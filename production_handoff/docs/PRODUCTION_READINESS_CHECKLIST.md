# Production readiness checklist

## 1. Архитектура
- [ ] Разделить монолитный App на компоненты.
- [ ] Вынести мок-данные в отдельный слой `mockData`.
- [ ] Подготовить слой API-клиента.
- [ ] Сделать единый store/state management для выбранных сущностей.

## 2. Интеграция с 1С / CRM
- [ ] Контрагенты.
- [ ] Пользователи и роли.
- [ ] Договоры.
- [ ] Счета.
- [ ] Оплаты.
- [ ] Акты / УПД / акт сверки.
- [ ] Заказы / отгрузки.
- [ ] Обращения.
- [ ] Базы 1С.
- [ ] Лицензии.
- [ ] Серверы / RDP.

## 3. Безопасность
- [ ] Авторизация.
- [ ] Сессии.
- [ ] Разграничение прав по контрагентам.
- [ ] Запрет доступа к чужим документам.
- [ ] Маскирование паролей.
- [ ] Логирование действий.

## 4. Платежи
- [ ] Оплата картой.
- [ ] СБП.
- [ ] Списание бонусов.
- [ ] Списание баланса.
- [ ] Частичная оплата.
- [ ] Скачивание счета.
- [ ] Привязка оплаты к заказу/документу.

## 5. Документы
- [ ] Скачать счет.
- [ ] Скачать УПД.
- [ ] Скачать акт.
- [ ] Заказать акт сверки.
- [ ] Выбор периода через календарь.
- [ ] Отправка на email.

## 6. UI/UX
- [ ] Проверить desktop 1440/1280.
- [ ] Проверить tablet.
- [ ] Проверить mobile 390/375/360 px.
- [ ] Проверить длинные формы.
- [ ] Проверить модальные окна.
- [ ] Проверить все состояния: hover, active, disabled, loading, error.

## 7. Приемка
- [ ] Нет мертвых кнопок.
- [ ] Все основные действия открывают нужную форму.
- [ ] Данные не смешиваются между разделами.
- [ ] Базы 1С не содержат серверных характеристик.
- [ ] Серверы/RDP отдельно от баз 1С.
- [ ] Товары и услуги разделены.

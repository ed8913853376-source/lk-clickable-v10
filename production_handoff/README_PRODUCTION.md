# Личный кабинет — production handoff package

Этот архив предназначен не только для просмотра прототипа, но и для передачи разработчикам.

## Важно

Последние версии после v10.17 частично правились в легком preview HTML. Поэтому пакет содержит два слоя:

1. `preview/index.html` — актуальный визуально-кликабельный прототип v11.2 для проверки UX.
2. `frontend-source/` — последняя полная React/Vite-структура проекта с исходниками, `package.json`, `src`, стилями и документацией.

Перед промышленным внедрением разработчик должен перенести все изменения из актуального preview в React-компоненты исходного проекта. Не внедрять напрямую одиночный HTML как финальный production-код.

## Быстрый просмотр

Откройте:

```text
preview/index.html
```

или:

```text
lk_client_portal_preview.html
```

## Запуск исходников

```bash
cd frontend-source
npm install
npm run dev
npm run build
```

`node_modules` в архив не включается специально.

## Что должно быть сделано перед production

1. Перенести актуальные изменения v10.18–v11.2 из `preview/index.html` в `frontend-source/src/app/App.tsx` и стили.
2. Разбить большой App на компоненты:
   - Sidebar
   - Dashboard
   - Billing
   - Documents
   - Orders
   - Tickets
   - Projects
   - Bases1C
   - Licenses
   - Modals
3. Заменить мок-данные на API.
4. Подключить авторизацию и права доступа.
5. Подключить 1С/CRM как источник данных.
6. Подключить оплату, СБП, счета и документы через реальные API.
7. Добавить тестирование основных сценариев.
8. Провести mobile QA.

## Статус

Это production-handoff package, но не финальный production build. Он пригоден для передачи программистам как база для промышленной реализации.

# Задача для проверки в ChatGPT

Проверь текущую версию кликабельного прототипа личного кабинета B2B IT-компании: `lk_clickable_v10`.

Контекст:
- Это прототип личного кабинета клиента IT-компании.
- Компания сопровождает 1С, автоматизирует бизнес, сдает серверы/RDP/VPS, обслуживает базы 1С, продает товары, выставляет счета/акты/КП, ведет заказы и доставки.
- Реальные API, 1С, Битрикс, платежи и скачивания файлов НЕ подключать.

Что проверить:
1. Все кнопки и кликабельные элементы должны открывать форму по смыслу раздела.
2. Не должно быть неподходящих универсальных форм.
3. Контрагенты, базы 1С, серверы, товары, услуги, финансы, лицензии и документы не должны смешиваться.
4. В форме базы 1С не должно быть CPU/RAM/SSD/IP/тарифа сервера.
5. В форме контрагента не должно быть полей базы 1С, сервера, лицензий или товара.
6. AnyDesk/RuDesktop должны открывать модалку `Удаленный доступ`.
7. RDP-клиент должен открывать профильное окно RDP.
8. После отправки формы должен быть прототипный успешный статус.
9. Главная должна быть рабочей сводкой без рекламных баннеров.
10. Шрифты и левое меню должны оставаться читаемыми.

Главные файлы:
- `src/app/App.tsx` — основной интерфейс и формы.
- `ACTION_MAP.md` — карта действий.
- `FORMS_SPEC.md` — спецификация форм.
- `UX_FLOWS.md` — пользовательские сценарии.
- `TODO_QUESTIONS.md` — вопросы для уточнения.
- `BUILD_NOTES.md` — ограничения сборки.
- `lk_client_portal_preview.html` — однофайловый визуальный предпросмотр.

Нужно:
- Найти ошибки UX/логики форм.
- Предложить точечные исправления.
- Не ломать текущий дизайн.
- Не уменьшать шрифты.
- Не подключать реальные интеграции.

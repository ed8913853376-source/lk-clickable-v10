# UX_FLOWS v10

## Общая логика
1. Пользователь нажимает кнопку в разделе.
2. Система определяет бизнес-смысл действия.
3. Открывается модальное окно только этого раздела.
4. Пользователь заполняет поля.
5. После отправки показывается прототипный статус.

## Быстрые действия главной
- Создать обращение -> Новое обращение -> Заявка создана.
- Заказать услугу -> Заказ услуги -> Заявка создана.
- Заказать товар -> Заказ товара -> Заказ сформирован.
- Счет на оплату -> Счет на оплату -> Запрос отправлен.
- Скачать AnyDesk / RuDesktop -> Удаленный доступ -> Запрос отправлен.
- Скачать тонкий клиент 1С -> Тонкий клиент 1С.
- Заказать сервер / RDP -> Сервер или RDP-клиент.

## Разделение сущностей
Контрагенты не используют поля баз 1С. Базы 1С не используют CPU/RAM/SSD/IP. Серверы не используют поля товара. Финансы разделены на разные формы: карта, счет, привязка карты, обещанный платеж.

## Прототипные ограничения
Скачивания, оплаты, API 1С и Битрикс не выполняются. Кнопки показывают прототипный успешный статус, чтобы согласовать UX перед разработкой.

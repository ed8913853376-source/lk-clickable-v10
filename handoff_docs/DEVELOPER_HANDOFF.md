# Передача программистам

## Что передается

- `index.html` — кликабельный HTML-прототип.
- `docs/ACTION_MAP.md` — карта действий.
- `docs/FORMS_SPEC.md` — спецификация форм.
- `docs/API_INTEGRATION_DRAFT.md` — черновик интеграции с 1С/CRM.
- `docs/IMPLEMENTATION_PLAN.md` — план внедрения.
- `docs/CHANGELOG.md` — история изменений.

## Как использовать

Прототип нужен как эталон UX и бизнес-логики. Его можно использовать для:

- оценки разработки;
- постановки задач frontend/backend;
- согласования с руководством;
- демонстрации клиентского сценария;
- подготовки ТЗ на интеграцию с 1С и Битрикс.

## Что нельзя считать готовым

- Нет реальной авторизации.
- Нет реальных API.
- Нет подключения к 1С.
- Нет реальных платежей.
- Нет реальных файлов документов.
- Нет защиты прав доступа.

## Рекомендация по реализации

Не внедрять этот HTML напрямую в боевой Битрикс как есть. Правильнее использовать его как прототип и перенести в нормальную архитектуру:

- компоненты frontend;
- backend API;
- интеграция с 1С;
- права доступа;
- журналирование;
- обработка ошибок;
- тесты.
